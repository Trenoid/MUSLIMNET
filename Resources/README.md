# Каталог ресурсов — документация

Один файл, из которого понятно: что это за система, как ею пользоваться, как добавлять ресурсы и таксономию, и как настроить Base в Obsidian.

---

## 1. Что это и зачем

**Каталог ресурсов** — это структурированный список материалов (книги, видео, курсы, статьи и т.д.) в виде заметок Obsidian. Один ресурс = одна заметка в папке `Resources/items/`. Вся классификация держится на:

- **Properties (YAML)** в начале каждой заметки;
- **внутренних ссылках** на справочники типов, категорий и подкатегорий;
- **таблицах и фильтрах** через Bases и Dataview.

Так удобно пополнять каталог нескольким людям, фильтровать по типу/категории и потом выгружать данные на сервер (всё остаётся в обычном Markdown).

---

## 2. Структура папок

```
Resources/
├── README.md              ← вы здесь (документация)
├── Resources Home.md      ← главная страница каталога (стартовая точка)
├── items/                 ← все ресурсы: одна заметка = один ресурс
├── taxonomy/              ← справочники (типы, категории, подкатегории)
│   ├── types/             ← тип ресурса: Книга, Видео, Статья, Курс...
│   ├── categories/        ← категория: Программирование, Ислам, Медицина...
│   ├── subcategories1/    ← подкатегория 1: Backend, Aqidah, Анатомия...
│   └── subcategories2/    ← подкатегория 2: Python, Хадисы, Нервная система...
├── views/                 ← страницы с таблицами (Dataview) и отчётами
├── templates/             ← шаблоны для новых заметок
├── fileclasses/           ← fileClass для Metadata Menu (выбор только из таксономии)
└── attachments/           ← вложения к ресурсам (скриншоты, файлы)
```

**С чего начать:** откройте **Resources Home.md** — там ссылки на все разделы и «Добавить ресурс».

---

## 3. Как связаны сущности

- **Ресурс** — заметка в `items/`. В свойствах указывает:
  - **type** → ссылка на заметку из `taxonomy/types/` (Книга, Видео...);
  - **category** → ссылка на заметку из `taxonomy/categories/` (Программирование, Ислам, Медицина...);
  - **subcategory_1** → ссылка на заметку из `taxonomy/subcategories1/` (Backend, Aqidah, Анатомия...);
  - **subcategory_2** → ссылка на заметку из `taxonomy/subcategories2/` (Python, Хадисы...).

- **Иерархия таксономии:**
  - У **категории** родителя нет (верхний уровень).
  - У **подкатегории 1** в свойстве `parent` — ссылка на категорию (например `[[Программирование]]`).
  - У **подкатегории 2** в `parent` — ссылка на подкатегорию 1 (например `[[Backend]]`).

- **Правило:** не придумывать значения «с клавиатуры». Тип, категория и подкатегории всегда выбираются из уже созданных заметок-справочников. Так не будет дублей и опечаток, и сервер сможет однозначно парсить данные (у каждой таксономии есть поле `slug`).

## 4. Как добавить новый ресурс

1. Откройте **Resources Home** → ссылка «Добавить ресурс (открыть шаблон)» ведёт на шаблон.
2. В Obsidian: **Core plugin «Templates»** должен быть включён. В настройках Templates укажите папку шаблонов: `Resources/templates`, и шаблон по умолчанию (или по горячей клавише): `Ресурс - шаблон.md`.
3. Создайте **новую заметку** в папке `Resources/items/`. Имя файла = название ресурса или его нормализованный вариант (например `Django.md`, `Курс по фикху.md`).
4. Вставьте шаблон: команда «Templates: Insert template» → выберите **Ресурс - шаблон** (или настройте автовставку для папки `items/`).
5. Заполните:
   - **title** — название ресурса;
   - **type** — замените ссылку на нужный тип из `taxonomy/types/` (например `[[Книга]]`, `[[Видео]]`);
   - **category** — ссылка на категорию из `taxonomy/categories/`;
   - **subcategory_1**, **subcategory_2** — по необходимости, ссылки на заметки из `taxonomy/subcategories1/` и `subcategories2/`;
   - **source_url** — основная ссылка на материал;
   - **language** — например `ru`, `en`;
   - **status** — `new` → `checked` → `uploaded` или `archived`;
   - **description** — краткое описание;
   - секции «Кратко», «Что это», «Ссылки», «Комментарии», «Связанные ресурсы» — по желанию.
6. Сохраните. Заметка попадёт в таблицы Dataview и в Base (если Base настроена на папку `Resources/items/`).

**Важно:** все новые записи создавайте только из шаблона, чтобы у всех ресурсов был одинаковый набор полей и секций.

### 4.1. Ограничение выбора типа, категории и подкатегорий (Metadata Menu)

Без плагина при вводе `type`, `category`, `subcategory_1`, `subcategory_2` Obsidian предлагает **все** ссылки хранилища (шаблоны, другие ресурсы, папки). Чтобы в подсказках были **только** нужные значения — заметки из папок таксономии — используется **Metadata Menu**.

**Что сделано в хранилище:**

- В папке `Resources/fileclasses/` создан fileClass **resource** (`resource.md`). В нём для полей `type`, `category`, `subcategory_1`, `subcategory_2` задан тип **File** с запросом Dataview: в выпадающий список попадают только файлы из папок `Resources/taxonomy/types`, `categories`, `subcategories1`, `subcategories2` соответственно. Поля `status` и `language` заданы как **Select** с фиксированным списком значений.
- В шаблоне ресурса в frontmatter добавлено свойство **fileClass: resource**, чтобы новые заметки сразу подхватывались этим fileClass.

**Что сделать в Obsidian:**

1. **Установить плагин Metadata Menu**
   - Настройки → Community plugins → Browse → найти «Metadata Menu» → Install → Enable.

2. **Указать папку fileClass**
   - Настройки → Community plugins → Metadata Menu → **Class Files Path** (или «FileClass files folder») → укажите `Resources/fileclasses`. Сохраните.

3. **Привязать fileClass к папке с ресурсами**
   - Откройте заметку `Resources/fileclasses/resource.md`.
   - В заголовке вкладки или рядом с названием файла появится кнопка (иконка) плагина. Нажмите её.
   - Откроется вид «Fileclass Settings». Найдите опцию **«Files paths option»** (или «Map with folder path»).
   - Добавьте путь: **Resources/items** (или `Resources/items/`). Сохраните.
   - Если в `resource.md` в frontmatter уже указано `paths: - Resources/items`, плагин может подхватить это автоматически; иначе задайте путь через эту опцию вручную.

4. **Проверка**
   - Откройте любую заметку из `Resources/items/` (например ресурс) и кликните в поле **type** (или category, subcategory_1, subcategory_2). Должен открываться выбор только из соответствующих папок таксономии (Видео, Книга для type; Программирование, Ислам, Медицина для category и т.д.), без шаблонов и прочих файлов.

Итог: при выборе типа — только заметки из `taxonomy/types/`, при выборе категории — только из `taxonomy/categories/`, подкатегории 1 — из `subcategories1/`, подкатегории 2 — из `subcategories2/`.

**Важно: откуда брать правильный список.** Панель «Свойства» (слева в Obsidian) при клике по полю со ссылкой показывает **стандартный** список ссылок всего хранилища — его плагин не меняет. Чтобы увидеть **только** типы/категории/подкатегории из таксономии:

1. **Кнопка Metadata Menu** — откройте заметку-ресурс, нажмите иконку плагина рядом с названием заметки (или с ссылкой на неё). Откроется модальное окно со всеми полями; у полей type, category, subcategory_1, subcategory_2 будет выбор только из нужных папок.
2. **Автодополнение в редакторе** — переключитесь в режим редактирования (исходный текст), в frontmatter поставьте курсор после `type:` (или другого поля) и введите **двоеточие** `:` — появится подсказка Metadata Menu с отфильтрованным списком.

Запросы Dataview в fileClass исправлены на формат `dv.pages('"путь/к/папке"')`, чтобы папки таксономии подхватывались корректно.

## 5. Как добавить новый тип ресурса

1. В папке `Resources/taxonomy/types/` создайте новую заметку, например `Статья.md`.
2. Можно взять за основу шаблон **Resources/templates/Тип ресурса - шаблон.md**: скопировать содержимое в новую заметку.
3. Заполните YAML:
   - `kind: taxonomy`
   - `taxonomy_type: type`
   - `parent:` (оставить пустым)
   - `slug:` — латиница для выгрузки на сервер, например `article`
4. Заголовок `# Статья` и секция «Описание» — по смыслу.
5. После сохранения новый тип можно выбирать в свойствах ресурсов (как ссылку `[[Статья]]`).

---

## 6. Как добавить новую категорию

1. Создайте заметку в `Resources/taxonomy/categories/`, например `Экономика.md`.
2. Используйте шаблон **Категория - шаблон.md**: `kind: taxonomy`, `taxonomy_type: category`, `parent:` пусто, `slug:` например `economika`.
3. Заголовок и описание — по смыслу.
4. При желании в ту же заметку можно добавить блоки Dataview «Ресурсы этой категории» и «По типам» — как в `Программирование.md` (скопируйте блоки и замените `[[Программирование]]` на `[[Экономика]]`).

---

## 7. Как добавить подкатегорию 1

1. Создайте заметку в `Resources/taxonomy/subcategories1/`, например `Frontend.md`.
2. Шаблон **Подкатегория 1 - шаблон.md**: `taxonomy_type: subcategory_1`, **parent** — ссылка на категорию, например `"[[Программирование]]"`, `slug:` например `frontend`.
3. Подкатегория 1 всегда принадлежит одной категории; у разных категорий могут быть свои подкатегории 1 (например у Ислам — Aqidah, у Медицина — Анатомия).

---

## 8. Как добавить подкатегорию 2

1. Создайте заметку в `Resources/taxonomy/subcategories2/`, например `JavaScript.md`.
2. Шаблон **Подкатегория 2 - шаблон.md**: `taxonomy_type: subcategory_2`, **parent** — ссылка на подкатегорию 1, например `"[[Frontend]]"`, `slug:` например `javascript`.
3. Подкатегория 2 всегда привязана к подкатегории 1 (и через неё — к категории).

---

## 9. Навигация по каталогу

- **Resources Home.md** — быстрые действия (добавить ресурс, новые, без категории, без ссылки, готово к выгрузке), ссылки на представления и таксономии.
- **views/** — страницы с таблицами Dataview:
  - **Все ресурсы** — полная таблица по `Resources/items/`;
  - **По программированию / По исламу / По медицине** — фильтр по категории + «Последние добавленные»;
  - **Проверка качества** — без типа, без категории, без source_url, дубликаты по названию, со статусом new, давно не обновлялись.
- **taxonomy/** — индексные страницы `types.md`, `categories.md`, `subcategories1.md`, `subcategories2.md` со ссылками на все справочники; плюс внутри категорий (например в `Программирование.md`) — таблицы ресурсов этой категории.

Закладки (Bookmarks) рекомендуется собрать в группы: Каталог (Resources Home), Представления (views + Base), Таксономии (папки taxonomy), Проверка качества, Выгрузка.

## 10. Создание Base «Все ресурсы» и представлений в Obsidian

Base — это встроенная в Obsidian возможность показывать папку с заметками в виде таблицы с фильтрами и сортировкой. Настраивается только через интерфейс. Ниже — пошагово.

### 10.1. Включить Bases

1. Откройте **Настройки** (Settings) — иконка шестерёнки в левом нижнем углу или `Ctrl+,`.
2. Слева: **Core plugins** → найдите **Bases** → включите тумблер.

### 10.2. Создать новую Base

1. В левой панели (обозреватель файлов) найдите папку **Resources**.
2. **Правый клик** по папке **items** → выберите пункт вроде **«New Base»** / **«Создать базу»** / **«Add as Base»** (формулировка зависит от версии Obsidian).
   - Либо: через команду Command Palette (`Ctrl+P`) введите **Base** и выберите действие создания новой Base, затем укажите папку `Resources/items`.
3. Откроется интерфейс Base. Если спросят «какую папку использовать» — укажите **Resources/items**.

### 10.3. Настроить колонки (столбцы)

В настройках Base (иконка шестерёнки или меню вида) откройте настройку **Columns** / **Колонки** и добавьте/упорядочьте поля в таком порядке:

| Порядок | Имя поля (property) |
|--------|----------------------|
| 1      | title                |
| 2      | type                 |
| 3      | category             |
| 4      | subcategory_1        |
| 5      | subcategory_2        |
| 6      | language             |
| 7      | status               |
| 8      | source_url           |
| 9      | updated              |

Имена должны совпадать с именами свойств в YAML заметок (без пробелов, строчными). Если какого-то столбца нет в списке — добавьте его (Add column / Добавить колонку) и выберите соответствующее свойство.

### 10.4. Сортировка по умолчанию

В настройках Base найдите **Sort** / **Сортировка**:

- Столбец: **updated**
- Порядок: **Descending** / по убыванию (новые сверху).

### 10.5. Фильтр по умолчанию (опционально)

Чтобы в базе по умолчанию показывались только ресурсы (а не случайные заметки без тега):

- Откройте **Filter** / **Фильтр** в Base.
- Добавьте условие: поле **tags** → **contains** / **содержит** → значение **resource**.

Сохраните настройки Base.

### 10.6. Создать представления (Views) внутри Base

В интерфейсе Base обычно есть переключатель представлений (вкладки или выпадающий список «View»). Создайте отдельное представление для каждого фильтра — так можно быстро переключаться между «все ресурсы», «по программированию» и т.д.

- **Все ресурсы**  
  - Фильтр: без дополнительных условий (или только `tags` содержит `resource`).

- **По программированию**  
  - Фильтр: **category** **is** / **equals** → `Программирование` (или ссылка `[[Программирование]]` — как Obsidian предложит для поля со ссылками).

- **По исламу**  
  - Фильтр: **category** → `Ислам` / `[[Ислам]]`.

- **По медицине**  
  - Фильтр: **category** → `Медицина` / `[[Медицина]]`.

- **Без заполненного URL**  
  - Фильтр: **source_url** **is empty** / **пусто**.

- **Без подкатегории**  
  - Фильтр: **subcategory_1** is empty **OR** **subcategory_2** is empty (если интерфейс позволяет «ИЛИ»; иначе два отдельных представления).

- **Готово к выгрузке**  
  - Фильтр: **status** **equals** → `uploaded`.

Имена представлений задайте так, как удобно (например те же, что выше). После сохранения переключение между вкладками/представлениями даёт нужные выборки без правки запросов.

### 10.7. Закрепить Base и представления в закладках

1. Включите Core plugin **Bookmarks**.
2. Откройте Base на папке `Resources/items` (или конкретное представление, если Obsidian даёт закрепить его).
3. Добавьте в закладки: пункт меню или кнопка «Add to bookmarks» / «Добавить в закладки».
4. В панели Bookmarks создайте группу, например **Представления**, и поместите туда ссылки на Base и на страницы из `Resources/views/` (Все ресурсы, По программированию и т.д.), а также **Resources Home** и **Проверка качества**.

После этого «как пользоваться» и «как добавить ресурсы/типы/категории» полностью описаны в этом README; настройка Base и закладок — один раз по этой инструкции.

## 11. Полезные плагины

- **Dataview** — уже используется в `views/` и в индексных страницах категорий; без него таблицы в этих заметках не заработают.
- **Templates** (Core) — вставка шаблона «Ресурс» при создании новой заметки в `items/`.
- **Metadata Menu** (Community) — **рекомендуется**: ограничивает подсказки для полей `type`, `category`, `subcategory_1`, `subcategory_2` только заметками из папок таксономии (см. разд. 4.1). Также даёт выпадающие списки для `status` и `language`. Требует установки и настройки папки fileClass и привязки к `Resources/items`.
- **Templater** — опционально, если нужна подстановка даты/автора и более сложная логика в шаблоне; для старта достаточно Core Templates.

---

## 12. Выгрузка на сервер

Данные готовы к парсингу: каждая запись — отдельный Markdown-файл с предсказуемым YAML. Ссылки на таксономии хранятся как `[[Тип]]`, `[[Категория]]` и т.д.; у каждой заметки в `taxonomy/` есть поле **slug** для маппинга на сервере (например `[[Видео]]` → `video`, `[[Программирование]]` → `programmirovanie`). Ресурсы лежат в `Resources/items/`, таксономия — в `Resources/taxonomy/`.

Если что-то осталось неясно — начните с **Resources Home.md** и одной заметки-ресурса по шаблону; дальше логика станет очевидной.
